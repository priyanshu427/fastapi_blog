# async used for concurrent requests where input/output is involved and we have cpu time being wasted. 
# sync is when we are waiting for a response from an external api or db , etc and no other task can be perfomed in the meanwhile.
# async isnt always faster as a complex calculation taking cpu load and trying to process another request slows down the app.
# a normal def function is sync , This means lines of code execute exactly one after another, and everything stops to wait for slow tasks. fastapi tucks it in a separate thread pool
# A Thread Pool is a collection of entirely separate, independent workers (threads) that the operating system spawns in your RAM. Each thread operates in its own little world, completely cut off from the others.
# a async def function runs directly in the main event loop when a task is not gonna block the cpu thread
# The Main Event Loop is a single, continuous, infinite while loop running on one single thread of your CPU. Its entire job is to look at a list of tasks, execute them if they are ready, and skip them if they are waiting on something slow (like a database read).
# cannot call sync methods with an async engine
# in sync lazy loading is used where if we need a object that is in a realtionship sqlachemy will run a hidden query for it which is sync.
# in async lazy loading will cause an error as it will try to run a traditional blocking sync query in the main event loop that will crash the app and error.
# for called objects we will use eager loading. here we give a unified db query that can fetch the objects at the same time
# in async even though the main event loop is a single high performance cpu thread it hands over the tasks like a sqllite query to the linux kernel which gives it the sqllite engine to execute it . in the meantime the cpu thread executes other requests.
# when sqlachemy stiches a object using two different tables using selectinload it checks the user id in the post table which is 1 and then locates the user_id in the user table. now it uses relationship to create a temporary instance in memory of post.author that can located by a pointer and contains the post and its author.
